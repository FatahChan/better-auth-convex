## `typedAdapter` on the component client

The recommended wrapper for typed adapter calls. `createClient` exposes
`typedAdapter` so you don't call
`ctx.runQuery(components.betterAuth.adapter.findOne, …)` directly — use
`authComponent.typedAdapter.findOne(ctx, …)` instead and the return type is
inferred from `model`.

With [local install](/features/local-install), pass your schema in
`createClient` config so types match your tables (including plugin tables):

```ts title="convex/auth.ts"
import { createClient } from "@convex-dev/better-auth";
import authSchema from "./betterAuth/schema";
import { components } from "./_generated/api";
import { type DataModel } from "./_generated/dataModel";

export const authComponent = createClient<DataModel, typeof authSchema>(
  components.betterAuth,
  { local: { schema: authSchema } }
);
```

```ts title="convex/users.ts"
import { query } from "./_generated/server";
import { v } from "convex/values";
import { authComponent } from "./auth";

export const getUserByEmail = query({
  args: { email: v.string() },
  handler: async (ctx, args) => {
    return await authComponent.typedAdapter.findOne(ctx, {
      model: "user",
      where: [{ field: "email", value: args.email }],
    });
    // Doc<"user"> | null
  },
});
```

## `createTypedAdapter()`

Lower-level helper when you need to build a typed wrapper yourself (same
methods as `typedAdapter`). `createClient` uses this internally.

```ts
import { createTypedAdapter } from "@convex-dev/better-auth";
import schema from "./betterAuth/schema";
import { components } from "./_generated/api";

export const adapter = createTypedAdapter(schema, components.betterAuth.adapter);
```

The adapter exposes the same operations as
[`createApi`](/features/local-install#export-adapter-functions): `findOne`,
`findMany`, `create`, `updateOne`, `updateMany`, `deleteOne`, and `deleteMany`.

### Exported types

| Type | Description |
| --- | --- |
| `TypedAdapter<S>` | Return type of `createTypedAdapter` for schema `S` |
| `AdapterFunctions` | Function references passed as the second argument |
| `AdapterWhere<S, T>` | Where-clause for table `T` in schema `S`; `field` and `value` narrow with the model |
| `SortBy` | `{ direction: "asc" \| "desc"; field: string }` for `findMany` |
| `BatchResult` | Paginated batch result from `updateMany` and `deleteMany` |

## `requireRunMutationCtx()`

A type guard to ensure a given context object has a `runMutation` property.
Requires a
[mutation context](https://docs.convex.dev/functions/mutation-functions#mutation-context)
or [action context](https://docs.convex.dev/functions/actions#action-context).
Will error if a
[query context](https://docs.convex.dev/functions/query-functions#query-context)
is provided.

## `requireActionCtx()`

A type guard to ensure the ctx is an action ctx. Will error if the provided ctx
is not an action ctx.

## Example usage

```ts
import { requireActionCtx } from "@convex-dev/better-auth/utils";
import { type GenericCtx } from "@convex-dev/better-auth";
import { Resend } from "@convex-dev/resend";
import { components } from "./_generated/api";
import { type DataModel } from "./_generated/dataModel";

export const resend = new Resend(components.resend);

export const createAuthOptions = (ctx: GenericCtx<DataModel>) => ({
  baseURL: siteUrl,
  sendVerificationEmail: async ({ user, url }) => {
    // This function only requires a `runMutation` property on the ctx object,
    // but we'll make sure we have an action ctx because we know a network
    // request is being made, which requires an action ctx.
    await resend.sendEmail(requireActionCtx(ctx), {
      to: user.email,
      subject: "Verify your email",
      html: `<p>Click <a href="${url}">here</a> to verify your email</p>`,
    });
  },
});
```
